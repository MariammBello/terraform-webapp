01

Initialize Terraform Project

Prerequisites

  • AWS Account: Required to create resources (EC2, S3, IAM, etc)
  • Ensure Terraform is installed: Install Terraform
  • IAM User with Terraform Permissions: A single IAM user policy grants permissions to manage EC2, S3, and IAM resources.
  • AWS CLI Configuration: Ensuring the IAM user's access and secret keys created above are configured locally to authenticate Terraform commands.
  • SSH Key Pair: Create a key pair to SSH into the EC2 instance. Use ssh-keygen to generate the keys: ssh-keygen -t rsa -b 2048 -f {replace with name of key}
  • Clone the repository: Terraform Web Deployment
  • Take a good shot at reading the readme.md documentation in the repo: Terraform Web Deployment

Once all the requirements are made, and you decide to clone this repo, replace the index.html, style.css and script.js to your preffered files.

02

IP & Domain Configuration (Optional)

1. Deploy Elastic IP

This deploys a static domain that remains constant for your domain

terraform apply -target=aws_eip.web_eip

2. Get Public IP

This outputs the Elastic IP that will be linked to your DNS record if you own one 

terraform output public_ip

3. DNS Configuration (Optional)

Follow these steps in your domain registrar:

  1. Log in to your domain provider's DNS settings
  2. Create a new A record:
    • Type: A
    • Name: @ (or www for subdomain)
    • Value: [Elastic IP from output]
    • TTL: 3600 seconds (recommended)
Allow 15-30 minutes for DNS propagation before proceeding.
03

Deploy Full Infrastructure

Project Initialisation

This command initializes Terraform, sets up the required backend, and downloads necessary provider plugins.

terraform init"

Option 1: With Custom Domain

terraform apply -var="domain_name=example.com"

Option 2: Without Custom Domain

terraform apply

The Deployment steps implements the following:

  • Creates and configures the EC2 instance with Ubuntu as the base image
  • Sets up security groups to configure security rules for allowing web and ssh traffic
  • Installs and configures Nginx web server
  • Configures SSL certificates automatically
  • Deploys your website files and saves them on an S3 bucket storage
  • Starts up the web service automatically
Please note that applying Terraform should be done using best practices, e.g., "Terraform plan" is necessary to run to evaluate what the actual implementation will do. See my Terraform blog on getting started with Terraform.

B2

Webserver setup

SSH into Instance

# SSH into your EC2 instance
ssh -i "your-key.pem" ubuntu@your-ec2-ip
# Install and configure web server
sudo apt update
sudo apt install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx
Deploy Frontend Files on the virtual machine 
# SSH into your EC2 instance
                                ssh -i "your-key.pem" ubuntu@your-ec2-ip
                                # Install and configure web server
                                sudo apt update
                                sudo apt install nginx -y
                                sudo systemctl start nginx
                                sudo systemctl enable nginx
3> Deploy Frontend Files on the virtual machine 
scp -i "your-key.pem" {index.html,style.css,script.js} ubuntu@your-ec2-ip:/var/www/html/
scp -i "your-key.pem" -r gifs/ ubuntu@your-ec2-ip:/var/www/html/
sudo systemctl restart nginx

01

Infrastructure Code Setup with Bash, Terraform and AWS S3

Get started with the infrastructure code:

Prerequisites

  1. Ensure Terraform is installed: Install Terraform
  2. Purchase a domain name: Google Domains or Namecheap
  3. Clone the repository: Terraform Web Deployment
  4. Ensure you update frontend files (index.html, style.css, script.js) to preference in the same directory
02

IP & Domain Setup

Set up your static IP and configure your domain:

Domain Configuration Process

# Step 1: Create and get Elastic IP
terraform init
terraform apply -target=aws_eip.web_eip
terraform output public_ip

# Step 2: Configure Domain DNS
# Go to your domain provider and add an A record:
# Type: A
# Name: @ (or subdomain)
# Value: [Your Elastic IP from above]
# TTL: 3600 (or as preferred)

What's happening?

  1. Creating a static IP that will remain constant for your domain
  2. Linking your domain name to this IP address

Wait for DNS propagation (usually 15-30 minutes) before proceeding to the next step.

03

Deploy Full Infrastructure

Deploy your complete infrastructure with SSL:

# Deploy the full infrastructure
terraform apply

This command will:

  • Create and configure the EC2 instance with Ubuntu as the base image
  • Set up security groups to configure security rules for allowing web and SSH traffic
  • Install and configure Nginx automatically
  • Deploy your website files to the Nginx webserver
  • Configure SSL certificates automatically
  • Start the web service automatically
04

Provisioning the server Manually

  1. Sign up for an AWS account
  2. Navigate to EC2 Dashboard
  3. Launch a new EC2 instance
  4. Configure Security Groups
  5. Navigate to acquire instance details
B2

Webserver setup

SSH Into Your Server

# SSH into your EC2 instance
ssh -i "your-key.pem" ubuntu@your-ec2-ip

# Install and configure web server
sudo apt update
sudo apt install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx

Deploy Frontend Files on the virtual machine

scp -i "your-key.pem" {index.html,style.css,script.js} ubuntu@your-ec2-ip:/var/www/html/
scp -i "your-key.pem" -r gifs/ ubuntu@your-ec2-ip:/var/www/html/
sudo systemctl restart nginx
B3

Verify Deployment

Access your website:

  1. Get your EC2 instance's public IP from AWS Console
  2. Open your browser and visit: http://YOUR_EC2_IP
  3. Verify all elements are loading correctly
  4. Check if GIFs are displaying properly
